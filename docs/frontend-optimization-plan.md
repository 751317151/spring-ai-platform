# 前端整体优化方案

## 目标

本轮优化围绕三个核心目标展开：

1. 让用户进入系统后更快到达主任务区域。
2. 减少全局导航、页面说明和辅助功能带来的视觉负担。
3. 保持现有功能可用前提下，优先收口布局和交互层级。

## 当前主要问题

### 1. 全局框架信息重复

- 左侧菜单、顶部工具栏、最近访问、快捷入口存在重复表达。
- 页面主内容还没开始，用户已经要先理解很多“系统级功能”。

### 2. 默认首页不够聚焦

- 当前首页更像监控总览，而不是高频工作入口。
- 对个人项目来说，最常用的是 AI 助手和知识库，不是图表和统计卡片。

### 3. 知识库主流程仍不够直接

- 列表页信息偏说明化。
- 详情页虽然已经改成左右布局，但顶部和侧栏仍占用了较多注意力。

### 4. 聊天页虽然已做减法，但外围壳层仍较重

- 全局顶部和左侧菜单仍在抢占空间。
- 用户进入系统后的第一感知仍然偏“后台管理系统”，不是“AI 工作台”。

## 本轮落地原则

### 全局布局

- 侧边栏只保留核心导航和必要的管理入口。
- 顶栏只保留页面标题、全局搜索、登录用户和退出。
- 删除最近访问、通知中心、快捷键说明等非核心干扰项。

### 首页

- 将首页收口为“工作入口页”。
- 核心任务优先展示：AI 助手、知识库、学习中心。
- 监控类信息只保留简洁摘要，不再堆叠完整图表和复杂表格。

### 知识库

- 列表页以“选择知识库并进入问答”为中心。
- 详情页把问答区域作为绝对主体。
- 文档上传与文档列表收纳进右侧工具面板，默认不抢主视线。

### 路由

- 系统默认进入 `/chat`，让主流程成为默认入口。
- 首页保留，但从“默认第一页”降为“工作入口页”。

## 本轮实施范围

- `web/src/router/index.ts`
- `web/src/layouts/MainLayout.vue`
- `web/src/components/common/AppSidebar.vue`
- `web/src/components/common/AppHeader.vue`
- `web/src/views/DashboardView.vue`
- `web/src/views/RagView.vue`
- `web/src/views/RagDetailView.vue`

## 暂不处理

- 不大拆 `ChatView.vue` 的业务逻辑。
- 不重做聊天消息、输入框、会话状态机。
- 不改后端接口。

## 预期结果

- 登录后默认直达 AI 助手。
- 整体界面更像“工作台”，不再像层级过深的后台系统。
- 知识库路径更清晰：先选库，再进入详情，再问答。
- 后续如果继续优化，下一阶段再专门拆分聊天页内部结构。
