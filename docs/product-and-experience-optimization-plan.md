## 目标

这份文档用于记录当前项目在“功能完善”“页面交互优化”“工程治理提效”三个方向上的后续优化点。
它不是对已完成事项的重复记录，而是面向下一阶段产品化、稳定性和体验升级的执行清单。

## 当前判断

从当前代码结构和页面能力看，项目已经具备较完整的平台骨架：

- 后端服务边界清晰
  - `auth-service`
  - `agent-service`
  - `gateway-service`
  - `rag-service`
  - `monitor-service`
- 前端控制台已覆盖主要工作台
  - 登录
  - 智能助手聊天
  - 知识库与文档管理
  - 用户与权限管理
  - 模型网关
  - 运行监控
  - MCP 管理
- 工程基础能力已经具备
  - `env` 外置配置
  - JWT 黑名单
  - 刷新令牌
  - CI
  - 基础单元测试
  - RAG 文档状态机
  - 指标和告警基础能力

当前项目的主要短板已经不是“模块缺失”，而是以下三类问题：

1. 某些能力已经有雏形，但离真实可用还差最后一段闭环。
2. 前端页面信息密度较高，但很多交互还停留在“可用”而不是“顺手”。
3. 工程治理已经起步，但还缺少更完整的测试、契约、文档和数据生命周期设计。

## 一、后端功能完善

### 1. 聊天与会话能力

#### 当前状态

- 已支持多助手聊天
- 已支持会话记忆、会话切换、重命名、归档、置顶
- 已支持流式输出和会话导出

#### 建议完善

1. 增加会话级参数配置
   - 模型选择
   - 温度
   - 最大上下文长度
   - 是否启用知识库检索
   - 系统提示词模板
   - 当前已支持后端存取，会话级配置尚未真正下发到各 Agent 的运行时模型调用参数
2. 增加消息级操作
   - 重新生成
   - 继续生成
   - 基于某条消息创建分支会话
   - 引用上一条回答继续追问
3. 增加后端会话摘要能力
   - 自动会话命名
   - 会话摘要缓存
   - 长会话压缩
   - 当前已具备基础摘要缓存与长会话压缩
4. 增加会话检索能力
   - 按关键词检索历史会话
   - 按助手类型检索
   - 按时间范围检索
   - 当前已支持关键词、置顶、归档、时间范围、数量限制筛选

### 2. Agent 能力深化

#### 当前状态

- 已支持助手注册和路由
- 已支持权限检查
- 已支持基础多智能体能力

#### 建议完善

1. 增加工具调用审计
   - 调用了哪些工具
   - 工具输入参数
   - 工具返回结果摘要
   - 工具耗时
   - 成功或失败状态
2. 增加工具级权限控制
   - 不只按助手类型授权
   - 细化到工具级
   - 细化到数据源级
3. 增加多智能体执行轨迹
   - 任务如何拆分
   - 各子任务由谁执行
   - 汇总结论如何生成

### 3. RAG 能力补齐

#### 当前状态

- 已支持知识库管理
- 已支持文档上传、失败、重试、重建索引
- 已支持分块预览
- 已支持检索问答和证据展示
- 部分结构化文件仍是“存储成功但未解析”

#### 建议完善

1. 优先补齐结构化文件解析链路
   - `csv`
   - `json`
   - `xml`
2. 增加分块策略配置
   - 分块大小
   - overlap
   - 按文档类型选择不同分块器
3. 增加知识库版本与文档版本
   - 文档更新后保留版本
   - 可回溯历史版本
   - 支持版本差异比较
4. 增加检索效果评估能力
   - 命中率
   - 反馈统计
   - 低分证据占比
   - 问题改写建议
5. 增加知识库权限隔离
   - 部门级隔离
   - 用户级隔离
   - 共享知识库与私有知识库共存

### 4. 模型网关能力提升

#### 当前状态

- 已支持模型注册、场景路由、负载策略
- 已支持基础模型监控和前端管理

#### 建议完善

1. 增加动态健康检查
   - 模型连通性探测
   - 失败率过高自动降载
   - 自动切换备用模型
   - 当前已支持基于连续失败的临时降级和候选池回退
2. 增加熔断、限流、重试策略
   - provider 熔断
   - 场景限流
   - 请求重试
   - 超时降级
3. 增加成本统计
   - 每模型成本
   - 每场景成本
   - 每用户成本
4. 增加路由决策解释
   - 本次为何选择该模型
   - 是否命中降级链路
   - 是否触发权重调整
   - 当前接口已返回候选模型、健康候选、降级候选、选择原因和是否触发回退

### 5. 监控与运维能力

#### 当前状态

- 已支持总览、趋势、告警、慢请求、失败样本、证据反馈等视图

#### 建议完善

1. 增加时间范围筛选
   - 最近 1 小时
   - 最近 24 小时
   - 最近 7 天
   - 自定义时间段
2. 增加 traceId 钻取能力
   - 列表可直接跳详情
   - 查看单次链路完整上下文
3. 增加告警处理流转
   - 确认
   - 静默
   - 处理中
   - 处理备注
   - 已恢复
4. 增加成本与容量视角
   - 模型消耗
   - 用户消耗
   - 知识库查询量
   - 存储容量变化
5. 增加慢请求归因
   - 鉴权慢
   - 检索慢
   - 模型慢
   - 存储慢

### 6. 用户与权限能力

#### 当前状态

- 已支持登录、刷新令牌、用户管理、权限规则管理

#### 建议完善

1. 增加账户安全能力
   - 修改密码
   - 密码复杂度策略
   - 登录失败锁定
   - 首次登录强制改密
2. 增加用户生命周期管理
   - 启用
   - 禁用
   - 锁定
   - 过期
3. 增加权限模板
   - 按岗位或部门快速套用
   - 避免逐条手工配置
4. 增加菜单权限与接口权限联动
   - 前端菜单不可见
   - 后端接口仍需严格校验
